KNU — это минималистичный, параноидальный, монолитный user-space, написанный на Rust. Мы не используем стандартные библиотеки. Мы не используем `crt0`. Мы общаемся с ядром операционной системы **напрямую**, глядя ему прямо в глаза через `asm!("syscall")`.

##  Философия KNU

1. **Glibc — это прокладка для слабаков.** Зачем вам абстракции? Ядро Linux предоставляет вам 300+ системных вызовов. Будьте мужчиной, загружайте номер сисколла в регистр `rax` и дергайте прерывание.
2. **Нулевой оверхед.** Мы выкинули `rust_eh_personality`, разматывание стека и стартовые файлы. Если наша программа падает — она падает сразу и жестко в `SIGSEGV`, как дед завещал.
3. **Всё — это один бинарник.** Зачем нам тысяча утилит, если мы можем засунуть `kcat`, `kecho`, `kls` и `kcd` в один монолитный шелл `kbash`? BusyBox плачет от зависти.
4. **Безопасность памяти? Не слышали.** Весь наш код обернут в `unsafe { ... }`. Компилятор Rust ругается, бормочет что-то про лайфтаймы и владение, но мы заклеили ему рот изолентой. У нас есть голые указатели `*mut u8` и мы не боимся их использовать.

##  Что внутри этой коробки с черной магией?

Прямо сейчас KNU содержит в себе `kbash` — полноценную POSIX-совместимую (почти) оболочку, которая умеет:

* Гулять по файловой системе (`kcd`).
* Читать файлы напрямую с диска кусками по 4096 байт (`kcat`).
* Показывать содержимое папок с **ЦВЕТАМИ** (потому что мы параноики, но у нас есть чувство стиля) (`kls`).
* Создавать процессы через `fork` и замещать их через `execve`. Вы можете запускать из `kbash` любой мусор из вашей старой ОС (например, `nano` или `python`), пока мы не переписали его аналоги на KNU.

А еще у нас есть 31+ системный вызов. От `mmap` до `socket`. Мы можем написать на этом веб-сервер. Мы можем написать на этом аллокатор. Нас ничто не остановит.

## Как собрать ЭТО

Обычный `cargo build` подавится и умрет, попытавшись прилинковать C-шный мусор. Собирать строго по инструкции для элиты:

```bash
# 1. Скармливаем линкеру флаг, отключающий мусорные стартовые файлы:
mkdir -p .cargo
echo '[target.x86_64-unknown-linux-gnu]' > .cargo/config.toml
echo 'rustflags = ["-C", "link-arg=-nostartfiles"]' >> .cargo/config.toml

# 2. Компилируем шелл:
cargo build --bin kbash --target x86_64-unknown-linux-gnu --release

# 3. Радуемся крошечному бинарнику
strip target/x86_64-unknown-linux-gnu/release/kbash
