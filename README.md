readme_text = r"""# KNU: Kernel Not Unix (или KNU's Not Useless-shit)

[no_std: YES] [no_main: YES] [libc: DELETED] [unsafe: 100%]

Вы устали от того, что ваш "Hello World" весит 5 мегабайт? 
Ваша glibc жрет память и делает за вашей спиной тысячу ненужных вещей до вызова main? 
GNU превратился в раздутого монстра, а современные разработчики разучились писать код без npm install?

Добро пожаловать в KNU. 

KNU - это минималистичный, параноидальный, монолитный user-space, написанный на Rust. Мы не используем стандартные библиотеки. Мы не используем crt0. Мы общаемся с ядром операционной системы напрямую через asm!("syscall").

## Философия KNU

1. Glibc - это прокладка для слабаков. Зачем вам абстракции? Ядро Linux предоставляет вам 300+ системных вызовов. Загружайте номер сисколла в регистр rax и дергайте прерывание.
2. Нулевой оверхед. Мы выкинули rust_eh_personality, разматывание стека и стартовые файлы. Если наша программа падает - она падает сразу и жестко в SIGSEGV.
3. Всё - это один бинарник. Зачем нам тысяча утилит, если мы можем засунуть kcat, kecho, kls и kcd в один монолитный шелл kbash? 
4. Безопасность памяти? Весь наш код обернут в unsafe. У нас есть голые указатели *mut u8 и мы не боимся их использовать.

## Что внутри

Прямо сейчас KNU содержит в себе kbash - командную оболочку, которая умеет:

* Перемещаться по файловой системе (kcd).
* Читать файлы напрямую с диска (kcat).
* Показывать содержимое директорий (kls).
* Создавать процессы через fork и замещать их через execve. 

У нас реализовано более 30 системных вызовов: от работы с памятью (mmap) до сетевых сокетов. Этого достаточно для написания любого системного софта.

## Как собрать

Сборка производится без стандартных библиотек:

```bash
#  Настройка линкера
mkdir -p .cargo
echo '[target.x86_64-unknown-linux-gnu]' > .cargo/config.toml
echo 'rustflags = ["-C", "link-arg=-nostartfiles"]' >> .cargo/config.toml

#  Компиляция
cargo build --bin kbash --target x86_64-unknown-linux-gnu --release

#  Очистка бинарника
strip target/x86_64-unknown-linux-gnu/release/kbash

# запуск
./target/x86_64-unknown-linux-gnu/release/kbash.
